## Action Schema Path

Canonical repo-local action schema for this slice:

`openapi/science_monkeys_actions.openapi.yaml`

This schema is based on the Human-provided current Science Monkey GPT Action schema and adds the approved read/resume GET endpoints:

- `GET /v1/whoami`
- `GET /v1/capabilities`
- `GET /v1/show`
- `GET /v1/resume`
- `GET /v1/flows/{flow_ref}/resume`
- `GET /v1/flows/{flow_ref}/history`
- `GET /v1/flows/{flow_ref}/artifacts`
- `GET /v1/flows/{flow_ref}/latest`
- `GET /v1/flows/{flow_ref}/next`
- `GET /v1/flows/{flow_ref}/stop-conditions`

The schema intentionally does not expose:

- `/v1/science/share`
- `/v1/science/execute-experiment`
- Option C queue mutation routes
- Human advancement routes
- deployment/certification/promotion routes

## Required Operator Workflow

Before any Science GPT performs a write action, operators should use the read/resume surface:

1. `/whoami`
2. `/capabilities`
3. `/show`
4. `/resume`
5. `/next`
6. `/stop-conditions`
7. `/open-artifact` only when a specific governed artifact body is needed

If any read/resume command returns `UNKNOWN`, `NOT_FOUND`, `FORBIDDEN`, `AMBIGUOUS`, `ARTIFACT_CONTENT_POLICY_DENIED`, or unsafe-path output, the operator must stop and classify the problem before issuing a write route.

## Known Error Handling

Expected bounded failures:

- no active flow: return a non-mutating empty/UNKNOWN resume state
- unknown flow: return fail-closed not-found behavior
- ambiguous flow or artifact: return fail-closed ambiguity behavior
- unsafe artifact path: return `UNKNOWN_UNSAFE_PATH` or deny body read
- secret-like artifact body: return `403 ARTIFACT_CONTENT_POLICY_DENIED`
- unauthorized role: return explicit forbidden/denied behavior

No GPT may reconstruct denied artifact content from chat or memory.
